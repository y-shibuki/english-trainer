# 開発フェーズ計画

## 方針

- **FE → AI(Worker) → BE** の優先順位で進める
- API仕様書に基づいてFEをモックで先行開発し、ローカルで動作確認できる状態を早期に作る
- BEは最後に実装し、FEをリアルAPIに切り替える形で統合する

---

## フェーズ一覧

| フェーズ | 内容 | ゴール |
|---------|------|------|
| Phase 0 | ドキュメント整備 | API仕様が固まった状態 |
| Phase 1 | FEモック開発 | ローカルでFE全画面が動く状態 |
| Phase 2 | AI/Worker実装 | OpenAI連携でQuestion生成できる状態 |
| Phase 3 | BE実装 | FastAPI + DB + 全APIエンドポイント |
| Phase 4 | 統合 | FEをリアルBEに接続し全体動作確認 |

---

## Phase 1: FEモック開発

ローカル環境でFEが単体で動作する状態にする。BEは不要。

### 1-1. プロジェクト構成

```
frontend/
├── src/
│   ├── components/
│   ├── pages/
│   ├── mocks/          # MSWハンドラ
│   │   ├── handlers.ts
│   │   └── browser.ts
│   ├── api/            # APIクライアント関数
│   └── main.tsx
├── package.json
└── vite.config.ts
```

技術スタック:
- Vite + React + TypeScript
- shadcn/ui + Tailwind CSS
- MSW（Mock Service Worker）v2

### 1-2. MSWセットアップ

- `msw` をdevDependenciesに追加
- `public/mockServiceWorker.js` を生成（`npx msw init public/`）
- 開発環境のみMSWを起動するよう `main.tsx` に条件分岐を追加

### 1-3. モックハンドラ実装

API仕様書の全エンドポイントに対応するMSWハンドラを実装する。

| ハンドラ | レスポンス内容 |
|--------|-------------|
| `POST /api/v1/articles` | 固定のArticleオブジェクトを返す |
| `GET /api/v1/articles` | 固定のArticle一覧を返す |
| `GET /api/v1/articles/:id` | 固定のArticle詳細を返す |
| `GET /api/v1/articles/:id/generation-status` | 状態が変化するポーリングを模倣（呼び出し回数で状態遷移） |
| `GET /api/v1/questions` | 固定のQuestion一覧を返す |
| `GET /api/v1/questions/:id` | 固定のQuestion詳細を返す |
| `POST /api/v1/questions/:id/answer` | answerとcorrect_answerを比較して正誤を返す |

### 1-4. 画面実装

| 画面 | パス | 使用エンドポイント |
|-----|------|----------------|
| Article一覧 | `/` | `GET /articles` |
| Article登録 | `/articles/new` | `POST /articles` |
| Article詳細・生成状況 | `/articles/:id` | `GET /articles/:id`, `GET /articles/:id/generation-status` |
| 問題解答 | `/articles/:id/quiz` | `GET /questions?article_id=:id`, `POST /questions/:id/answer` |

### 確認ポイント

- Article登録 → 一覧に追加されることを確認
- 生成進捗が `pending → in_progress → completed` に遷移することを確認（モックで模倣）
- 単語問題・穴埋め問題それぞれの解答フローが動くことを確認
- 正解・不正解で解説が表示されることを確認

---

## Phase 2: AI/Worker実装

MSWモックと同じレスポンスをOpenAI APIで生成できる状態にする。

- Worker用Pythonエントリーポイント作成
- GenerationJobの取得・状態更新ロジック
- Sentence単位でOpenAI APIを呼び出し、Question（vocabulary / fill_in_the_blank）を生成
- 生成結果をDBに保存

この段階ではBEのHTTP APIは不要。WorkerはDBに直接書き込む。

---

## Phase 3: BE実装

FastAPIで全エンドポイントを実装する。

- Pythonプロジェクト構成・Dockerfileの作成
- Alembicでマイグレーション管理
- Article, Sentence, Question, GenerationJobのSQLAlchemyモデル定義
- 全エンドポイントの実装（API仕様書に準拠）
- Sentence分割ロジック（Article登録時に自動実行）

---

## Phase 4: 統合

FEのMSWモックをリアルBEに切り替えて動作確認する。

- FEの環境変数でベースURLを切り替え（`VITE_API_BASE_URL`）
- docker-composeでFE + BE + Worker + DBを一括起動
- Article登録からQuestion生成・解答までの全フローを確認
- Render + Neonへデプロイ
