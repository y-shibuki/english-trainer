# 開発フェーズ計画

## 方針

- **FE → AI(Worker) → BE** の優先順位で進める
- API仕様書に基づいてFEをモックで先行開発し、ローカルで動作確認できる状態を早期に作る
- BEは最後に実装し、FEをリアルAPIに切り替える形で統合する

---

## フェーズ一覧

| フェーズ | 内容 | ゴール |
|---------|------|------|
| Phase 0 | ドキュメント整備 | API仕様が固まった状態 |
| Phase 1 | FEモック開発 | ローカルでFE全画面が動く状態 |
| Phase 2 | AI/Worker実装 | OpenAI連携でQuestion生成できる状態 |
| Phase 3 | BE実装 | FastAPI + DB + 全APIエンドポイント |
| Phase 4 | 統合 | FEをリアルBEに接続し全体動作確認 |

## Phase 1: FEモック開発（完了）

ローカル環境でFEが単体で動作する状態にする。BEは不要。
FEの責務は「問題取得・表示・解答判定」のみ。Article登録UIや生成進捗表示は含まない。

### 1-3. セットアップ手順


### 1-4. 画面構成

| 画面 | パス | 使用エンドポイント |
|-----|------|----------------|
| 問題形式選択 | `/` | なし |
| クイズ | `/quiz?type=vocabulary` or `/quiz?type=fill_in_the_blank` | `GET /questions`, `POST /questions/:id/answer` |

### 1-5. MSWモックハンドラ

questionsエンドポイントのみ実装する。

| ハンドラ | レスポンス内容 |
|--------|-------------|
| `GET /api/v1/questions` | `type` と `limit` でフィルタした固定Question一覧 |
| `POST /api/v1/questions/:id/answer` | `correct_answer` と比較（lowercase + trim）して正誤を返す |

### 1-6. QuizPage の動作仕様

- 問題形式選択画面で選択した `type` で10問を取得
- 1問ずつ表示し、解答後に正誤と解説を表示してから次の問題へ
  - vocabulary: 英単語を提示し、日本語訳の選択肢4つをボタンで表示
  - fill_in_the_blank: 空欄を含む文を提示し、英単語の選択肢4つをボタンで表示
- 各問題の下部に出典（`source.title` をリンク表示、`source.url` があれば `<a>` に）
- 10問終了後: 正答率サマリーと「もう一度」ボタン

### 確認ポイント

- トップ画面で問題形式を選択 → クイズ画面に遷移することを確認
- vocabulary問題の選択肢をクリック → 正誤と解説が表示されることを確認
- fill_in_the_blank問題の選択肢をクリックして解答できることを確認
- 出典リンクが表示されることを確認
- 10問終了後に正答率サマリーが表示されることを確認
- ブラウザDevToolsのNetworkに `[MSW]` がついたリクエストが見えることを確認

## Phase 2: AI/Worker実装

MSWモックと同じレスポンスをOpenAI APIで生成できる状態にする。

- Worker用Pythonエントリーポイント作成
- GenerationJobの取得・状態更新ロジック
- Sentence単位でOpenAI APIを呼び出し、Question（vocabulary / fill_in_the_blank）を生成
- 生成結果をDBに保存

この段階ではBEのHTTP APIは不要。WorkerはDBに直接書き込む。

---

## Phase 3: BE実装

FastAPIで全エンドポイントを実装する。

- Pythonプロジェクト構成・Dockerfileの作成
- Alembicでマイグレーション管理
- Article, Sentence, Question, GenerationJobのSQLAlchemyモデル定義
- 全エンドポイントの実装（API仕様書に準拠）
- Sentence分割ロジック（Article登録時に自動実行）

---

## Phase 4: 統合

FEのMSWモックをリアルBEに切り替えて動作確認する。

- FEの環境変数でベースURLを切り替え（`VITE_API_BASE_URL`）
- docker-composeでFE + BE + Worker + DBを一括起動
- Article登録からQuestion生成・解答までの全フローを確認
- Render + Neonへデプロイ
