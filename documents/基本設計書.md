# 英語力向上アプリ 基本設計（複数プロセス構成）

## 1. プロダクト定義

本システムは、技術英語テキストを入力とし、AIを用いて英文法や英単語の問題および解説を非同期生成し、生成物を永続化・提供するWebアプリケーションである。

主目的は以下とする：

- AIを組み込んだプロダクト設計・実装体験
- 非同期処理を含むアーキテクチャ構築
- FE / BE / DB の分離
- 構造的に整理された設計の実現

---

## 2. アーキテクチャ構成

### 2.1 プロセス構成（複数プロセス）

本システムは以下の独立プロセスで構成する：

1. Frontend（UI）
2. Backend API
3. Worker（AI生成専用）
4. Crawler（データ取得専用）

各プロセスは独立して起動可能とする。

---

## 3. 責務分離

### 3.1 Frontend

- 問題形式選択
- 問題表示
- 解答入力・正誤判定
- 出典表示

### 3.2 Backend API

- Article登録
- Sentence分割
- GenerationJob作成
- Question取得API
- DB永続化

AI呼び出しは行わない。

### 3.3 Worker

- GenerationJobの監視
- AI呼び出し
- 問題生成
- 解説生成
- Question保存
- Job状態更新

ユーザーとの直接的なインターフェースは持たない。

### 3.4 Crawler

- 外部技術サイトからデータ取得
- Articleとして保存可能な形式に変換

自動実行は必須ではない。

---

## 4. 問題形式

### 4.1 単語問題（vocabulary）

英単語を提示し、その日本語訳（または英訳）を4択から選ぶ問題。

- 単語抽出はアプリ側ロジック
- 選択肢・意味・解説はAI生成
- 正解判定はアプリ側

### 4.2 穴埋め問題（fill_in_the_blank）

文章中の空欄に入る英単語を4択から選ぶ問題。

- SentenceをAIに入力
- 穴埋め文・選択肢・正解・解説はAI生成
- 正解判定はアプリ側

将来的に並び替え問題などを拡張していく。

---

## 5. 非同期処理モデル（バッチ型）

### 5.1 フロー
- Article登録
- Sentence分割
- Sentenceは「未生成」状態で保存
- Generatorバッチ実行
- 未生成Sentenceを取得
- AI実行
- Question保存
- 生成済みフラグ更新

---

## 6. データ思想

- Questionは生成後固定資産とする
- 再生成は行わない
- Sentenceとの参照関係のみ保持
- 生成物はimmutableとして扱う

---

## 7. レイヤ構造（Backend内部）

### Presentation Layer
- APIエンドポイント

### Application Layer
- ユースケース処理
- GenerationJob作成

### Domain Layer
- Article
- Sentence
- Question
- GenerationJob

### Infrastructure Layer
- Database
- AI Client
- 辞書API Client

依存方向は内向きのみとする。

---

## 8. スコープ外（初期段階）

- 難易度管理
- SRS
- 学習分析
- マルチユーザー対応
- リアルタイムAI生成

---

## 9. 設計原則

- 実装速度を最優先
- 過度な抽象化を避ける
- プロセス責務を明確に分離
- AIはWorker内に閉じる
- 将来拡張可能な構造を維持する

---

## 10. 成功定義

- プロセス分離が明確である
- 非同期処理が自然に組み込まれている
- AIがインフラ層に閉じている
- FE / BE / DB が分離されている
- 設計が一貫して整理されている
